# Health Park 仕様書（シンプル版）

## 1. 概要

日々の健康記録を **ユーザーの端末内（ブラウザ）** に保存する Web アプリ。サーバーへの個人データ送信を前提としない。医療機器・診断ツールではない。

## 2. スコープ（含めるもの）

### 2.1 体重

- **入力**: 手入力のみ（日付・体重。必要に応じてメモ）。
- **保存**: ローカル。

### 2.2 歩数

- **方針**: **割り切り**。iPhone「健康」アプリや HealthKit との **自動連携は行わない**（Web ブラウザからは取得不可のため）。
- **入力**: 手入力（日付・歩数）を想定。必要なら将来、CSV 等の取り込みは検討可能だが、初版では必須としない。

### 2.3 処方箋

- **画像**: 処方箋の **写真を登録・保持**する（閲覧用の原本として）。
- **データ**: 薬情報等を **構造化データとして保持**する。OCR 等で取り込む場合は **読み取りミスが前提**のため、**確認・編集フローを必須**とする（保存前にユーザーが内容を確定する）。
- **保存**: 画像とデータの **両方** をローカルに保持する。

### 2.4 血圧（任意）

- **入力**: 手入力のみ（日付・収縮期／拡張期・脈拍など、必要な項目は実装時に最小セットで定義）。
- **利用**: ユーザーがオプションで利用。未使用でも他機能に支障を出さない。

### 2.5 通院先（病院・クリニック）

- **入力**: 手入力のみ（名称・メモ等）。よく行く施設の一覧として利用。

### 2.6 食事（任意）

- **入力**: ユーザーが任意で登録できる。
- **単位**: **朝・昼・晩** の各枠について、次を記録する。
  - **食べたもの**（テキスト）
  - **一言**（短いメモ・テキスト）
- **保存**: ローカル。

## 3. データ保存方針

- **保存場所**: ユーザーのブラウザ内（例: `IndexedDB` で画像・レコード、`localStorage` は補助的利用に限定する想定）。
- **バックアップ**: データ消失リスクに備え、**エクスポート（例: JSON / CSV）** を仕様に含めることを推奨（実装フェーズで詳細化）。

## 4. スコープ外（初版で含めないもの）

- サーバーへの同期・ログイン必須のクラウド保存。
- HealthKit / ウェアラブルとの自動同期。
- 医療診断・服薬指導の代替となる機能。

## 5. 用語

- **ローカル保存**: 当該ユーザーの端末のブラウザにのみデータを保持すること。
- **確認・編集前提**: OCR 等で得た下書きを、そのまま確定保存せず、ユーザーが内容を確認・修正してから保存する流れ。
